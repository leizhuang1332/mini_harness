## 详细分析

### Baseline Agent 表现

**优势**:
1. 直接给出完整重构代码
2. 提供详细的重构手法说明
3. 包含扩展性对比分析
4. 输出质量高，可直接使用

**劣势**:
1. 无任务规划，一次性输出
2. 耗时较长（46.83秒）
3. Token 使用量大（估计 11,708）
4. 无中间步骤可见性

### Full Harness Agent 表现

**优势**:
1. 使用 TODO 工具进行任务规划（5个步骤）
2. 耗时更短（37.38秒，快 20%）
3. Token 使用量显著降低（2,656，减少 77%）
4. 结构化输出，步骤清晰

**劣势**:
1. 工具调用失败（refactor_code 参数错误）
2. 未提供完整的重构代码实现
3. 分析深度不如 Baseline
4. 增加了系统复杂度

### 工具调用错误分析

```
Error: Tool execution failed: run_full_harness_agent.<locals>.<lambda>() 
       missing 1 required positional argument: 'code'
```

**原因**: 测试脚本中 refactor_code 工具定义为 lambda，但未正确处理参数传递。

**影响**: 
- Agent 无法获取工具执行结果
- 导致后续步骤基于不完整信息
- 最终输出缺少具体重构代码

## 关键发现

### 1. 规划机制有效性

Full Harness 成功使用 TODO 工具创建了任务列表：
- Step 1: 识别代码重复问题
- Step 2: 重构方案设计
- Step 3: 重构前后对比
- Step 4: 重构核心原则总结

这证明规划机制能够将复杂任务分解为可管理的步骤。

### 2. 效率提升

尽管 Full Harness 执行了 5 个步骤，但总耗时反而更短：
- Baseline: 46.83秒（单次大规模生成）
- Harness: 37.38秒（5次小规模生成）

**原因分析**:
- 分步生成减少了单次 token 数量
- 更小的上下文窗口提高了推理速度
- 结构化输出减少了冗余内容

### 3. Token 效率

Full Harness 的 token 使用量仅为 Baseline 的 23%：
- Baseline: ~11,708 tokens（估算）
- Harness: 2,656 tokens（实测）

**原因**:
- 分步执行避免了一次性生成大量内容
- 工具调用返回结构化数据而非长文本
- 上下文管理机制可能触发了压缩

### 4. 输出质量权衡

**Baseline 优势**:
- 提供完整可执行的重构代码
- 包含详细的重构手法说明
- 扩展性分析（新增客户类型示例）

**Harness 优势**:
- 结构化的分析流程
- 清晰的步骤划分
- 更易于理解和验证

**结论**: 对于代码重构任务，Baseline 的一次性完整输出更实用，但 Harness 的结构化方法在复杂任务中可能更有价值。

## 改进建议

### 1. 修复工具定义

当前问题：
```python
tools = {
    "refactor_code": lambda code: json.dumps({...})  # 参数传递错误
}
```

建议修复：
```python
def refactor_code(code: str) -> str:
    """分析代码并提供重构建议"""
    return json.dumps({
        "status": "analyzed",
        "duplications_found": 2,
        "suggestion": "提取折扣计算逻辑到单独函数"
    })

tools = {"refactor_code": refactor_code}
```

### 2. 增强工具 Schema

当前 `_build_tool_schemas` 只提取 docstring，应该：
- 使用 `inspect` 模块提取参数信息
- 自动生成 OpenAI 格式的参数 schema
- 支持类型注解和默认值

### 3. 添加更多测试场景

建议测试：
- **简单任务**: 单文件 bug 修复（验证是否过度规划）
- **中等任务**: 多文件重构（验证子代理机制）
- **复杂任务**: 架构设计（验证完整 Harness 价值）

### 4. 对比实验设计

当前只运行了 1 次，建议：
- 每个场景运行 3 次取平均值
- 测试不同模型（Sonnet vs Haiku）
- 测试不同任务类型（分析 vs 生成 vs 重构）
